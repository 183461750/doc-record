git命令中的--recurse-submodules作用

在Git中，--recurse-submodules标志用于在克隆主项目的同时递归地克隆所有子模块。子模块是Git中一种将一个Git仓库作为另一个Git仓库的子目录的方式，通常用于当主项目需要依赖于其他项目时。
当你运行带有--recurse-submodules标志的git clone命令时，Git会执行以下操作：

  1. 克隆主项目仓库。
  2. 检测主项目中的.gitmodules文件，该文件包含了子模块的配置信息。
  3. 对于.gitmodules文件中列出的每个子模块，Git会克隆子模块的仓库到主项目中指定的路径。
  4. 将子模块注册到主项目的.git/config文件中，以便于后续的子模块操作，如更新、推送等。
    简而言之，--recurse-submodules的作用是确保在克隆主项目的同时，其依赖的所有子模块也被克隆并设置好，以便开发者可以直接开始工作，而无需手动初始化和更新子模块。
